- Added the **enforcement layer** for the Initiative tier (M1.2, milestone *v2 consuming initiatives*): three `pre_tool_use.sh` invariants make the tier coherent. (1) **Label mutual-exclusivity** — `gh issue edit <N> --add-label initiative` on a `directive` Issue (and `--add-label directive` on an `initiative` Issue) is blocked. (2) **Directive parent-XOR** — `--add-label directive` is blocked unless the body carries exactly one parent kind: a `## MISSION fit` field XOR a line-1 `Parent Initiative: #N` marker. Both fold into the existing `label-parent-consistency` matcher (`SKIP_HOOKS=label-parent-consistency`). (3) **`initiative-readonly`** (new matcher / escape category) — a mutating `gh issue edit`/`close`/`reopen` on an `initiative` Issue is blocked; `gh issue comment` is always allowed (the shell consumes Initiatives, never mutates them). All three fail open on unresolvable predicates. Adds the `issue_has_mission_fit_field` tri-state resolver. (#251)
