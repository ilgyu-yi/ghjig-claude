- Hardened the tier-3 branch-protection tool's (`scripts/install_branch_protection.sh`) 404 discrimination: all four HTTP-404 checks (the two verify GETs, the path-1 rulesets-list gate, and the path-2 current-protection GET) now match the structured HTTP-404 status phrasing via a single-sourced `_gh_err_is_404` helper instead of a bare `404` substring, so a non-404 (403/5xx) error whose message incidentally contains "404" is no longer mistaken for a genuine 404 — the fail-closed gates stay closed and verify stays `unreadable` rather than false-`absent`.
