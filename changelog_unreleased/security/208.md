- Fixed the trailing `# claude-eng:skip=` escape sentinel firing when the `#` is inside a quoted argument rather than a genuine unquoted shell comment: ordinary quoted text (a commit message, a PR-body paste, audit output quoting the sentinel) could silently set `SKIP_HOOKS` and disarm every matcher with a falsified audit reason while the shell ran the command intact. The sentinel is now honored only when its `#` is a real comment token, resolved with the same `python3`/`shlex` machinery the leading env-prefix uses; `python3` absent → sentinel not honored (fail-safe, enforcement stays armed).
