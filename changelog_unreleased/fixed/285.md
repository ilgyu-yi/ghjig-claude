Closed the reviewer working-tree-isolation gap (surfaced during the v0.3 run): the read-only-by-intent subagents (`code-reviewer`, `security-reviewer`, `issue-reviewer`, `plan-reviewer`, `activation-reviewer`, `explorer`) share the parent session's working tree and carry `Bash`, so a tree-mutating git command inside one (`git checkout <base> -- <path>`, `restore`, `stash`, `reset`, `add`) silently reverts or stages the parent's uncommitted work — observed once as a post-commit reversion of `scripts/test/smoke.sh`. Two layers now prevent it: (a) the SPEC §1.5 norm makes **worktree isolation** the canonical way to invoke these subagents, and (b) each such agent's prompt is constrained to **read-only git only** (compare a base via `git diff <base>...HEAD` / `git show <ref>:<path>`, never `git checkout <base> -- <path>`). smoke §77 asserts the constraint is present in all six agent files. The structural backstop (merges build from the pushed commit, not the working tree) means a post-commit reversion never ships; the norm adds "git status before each commit/merge" as the catch-all for the pre-commit window.
