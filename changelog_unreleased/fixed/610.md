- On a GHES target the `/file-review` review-posting path and the `merge-review` gate reading the review now pin `gh api` to the repo's host (derived from `gh repo view`), instead of gh's default host (github.com) — so the review actually lands on the target host, the merge gate can read it, and the acting-identity lookup stops mis-computing self-merge. `/file-review` URL validation is now host-agnostic (accepts a pull URL on any host).
