- The protected-branch push gate no longer false-blocks when a protected token (`main`/`master`/`release/*`) appears only inside a heredoc body or in a sibling non-push segment of a compound command (e.g. `git push origin <feat> && gh pr create --base main`). Detection now heredoc-strips the command (shared `strip_command_data` helper, factored from #340) and scans only the `git push` command segment(s); the same heredoc-strip is applied to the `git clean -f` gate. Quoted literals are deliberately not stripped on the push arm, so a genuine quoted target (`git push origin "main"`) still blocks. Fail-closed: genuine protected pushes (bare/`-u`/`HEAD:main`/env-prefixed/non-first-`&&`-segment/quoted-target) still block, and any strip failure falls back to the full-command match. (#366)
