- `onboard_target.sh` tier-3 install now detects an **untracked** `.github/` and opens the substrate PR instead of silently skipping it. The change-detection gate used `git diff --quiet -- .github/`, which is blind to untracked paths, so a greenfield target (no pre-existing `.github/`) got "already match (no PR needed)" and never installed the dir-mode workflows. Switched to `git status --porcelain -- .github/`. (#343)
