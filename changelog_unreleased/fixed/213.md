- Fixed the staged-secret commit check failing **closed** when `secret_scan.sh` is missing: an unguarded `scan_staged_secrets` call meant an undefined predicate (e.g. the session-restart helper-miss `safe_source` is designed to degrade gracefully) made `command not found` block **every** `git commit`, forcing `SKIP_HOOKS=secret` on each one — the opposite of the SPEC §6.1 fail-policy table's documented fail-open. The call is now guarded with `command -v` like its sibling matchers, so a missing helper skips the secret arm (fail-open) while `safe_source` still logs the `helper-missing` warn. (#213)
