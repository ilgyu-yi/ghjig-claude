Closed three selector-normalization bypasses in the dir-mode enforcement matchers (audit Theme A, EI of Directive #275). The `label-parent-consistency` matcher now resolves its `gh issue edit … --add-label` selector via a shared `resolve_gh_issue_target` helper that is tolerant of selector **form** (bare / quoted / **URL** — A1, previously a silent bypass) and flag **order** (the positional issue arg need not immediately follow `edit` — A2), and threads a `-R`/`--repo` (or URL-embedded) **target repo** into the marker/label predicates so a cross-repo relabel resolves against the command's actual repo (A3); the predicates' new optional `repo` arg also keys their per-session cache by `owner/name`, so a foreign `#N` lookup can't poison the current repo's same-numbered entry. `initiative-readonly` migrates to the same helper (gaining flag-order tolerance) but deliberately stays **current-repo-only** — cross-repo Initiatives are out of scope (§1.7), so it governs only same-repo Initiatives. `trusted-filer-mutate`'s own parser is untouched.
