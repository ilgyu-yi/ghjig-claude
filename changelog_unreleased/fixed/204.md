- Scoped the `force-push` hook matcher so it no longer blocks force-push unconditionally: force-push to an explicitly-named non-protected branch (`git push --force-with-lease origin <branch>`) is now allowed — the normal tail of the rebase-pull flow (SPEC §13) — while a force-push naming a protected branch (`main`/`master`/`release/*`), or a bare/remote-only force-push with no explicit target, is blocked. The bare form is blocked rather than inferred from the current branch because a no-refspec push's real destination is config-dependent (`push.default`/upstream), so the branch name isn't a safe proxy; the block message tells you to name the target. Escape: `SKIP_HOOKS=force-push`. (#204)
