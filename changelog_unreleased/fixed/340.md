- The `ac-closeout` and `merge-strategy` hook gates no longer false-trigger when `gh pr merge` appears only as command *data* — a heredoc body, a quoted argument, or a commit message — rather than as an actual command. Detection now strips heredoc bodies and quoted string literals before testing for the command words (shared `is_pr_merge_command` predicate), and stays fail-closed (any parse uncertainty, missing `python3`, or missing helper → still treated as a merge, so real merges are always gated).
